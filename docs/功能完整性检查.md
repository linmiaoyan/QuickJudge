## 二、前端模块与对接

### 1. 首页（tab-home）
- 入口卡片：文件管理、智能阅卷、班级中心、阅卷管理、布置练习
- `switchTab('home'|'files'|'ocr'|'center'|'grading'|'assign')` 均已绑定 ✅

### 2. 文件管理（tab-files）
- 选择扫描根目录、班级、日期文件夹，文件列表
- 选择文件、设置学号、识别、批阅
- 接口：/api/folders, /api/class_folders, /api/files, /api/ocr, /api/grade, /api/organize, /api/files/batch_delete, /api/file (DELETE), /api/scan_output_config, /api/scan, /api/import_local_images, /api/check_conversion, /api/check_new_files
- 选择答题卡 → 导入纸张（扫描仪/本地文件）→ 扫描仪高级设置 → 开始扫描 ✅

### 3. 智能阅卷（tab-ocr）
- 仅识别：`recognizeText()` → POST /api/ocr → 展示识别结果+译文 ✅
- 识别并批阅：`recognizeAndGrade()` → /api/ocr → /api/grade（不保存班级中心）✅
- 识别并批阅+班级报告：`recognizeAndGradeWithClassReport()` → /api/ocr → /api/grade（save_to_class_center: true）✅
- 学号映射弹窗、设置学号 ✅
- 批阅结果展示、结果区域与历史 ✅

### 4. 阅卷管理（tab-grading）
- 子视图：阅卷管理 / 我的阅卷；创建阅卷任务；阅卷进度
- `initGradingTab()`、`setGradingSubTab()`、`showGradingView()`、`loadGradingTaskListManage()`、`loadGradingTaskListMine()`、`loadGradingProgressTable()`、`syncGradingScores()` 等已实现
- 创建阅卷任务向导（两步：选择练习 + 阅卷设置）、批阅弹窗（打分板）
- **数据**：当前为前端 Mock，无后端接口；后续可接真实阅卷 API ✅

### 5. 班级中心（tab-center）
- `loadClassCenterView()` → GET /api/classes → 班级卡片列表 ✅
- `loadClassReports(className)` → GET /api/class_center/<class_name>/reports → 按日期报告列表 ✅
- `loadReportDetail(className, date)` → GET /api/class_center/<class_name>/reports/<date> → 班级评价 + 学生列表 ✅
- `loadStudentRecords(className, studentId)` → GET /api/class_center/<class_name>/<student_id> → 历次记录（含图片、作文原文、批阅报告）✅
- 作文原文依赖批阅时写入的 `essay_text`，已在后端补全 ✅

### 6. 布置练习（tab-assign）
- 练习类型、筛选、题目列表、试卷篮、导出试卷、答题卡设置（含打印说明）
- `initCompositionAssign()`、`loadCompositionExercises()`（Mock）、`loadCompositionReports()`（来自 /api/classes）
- **数据**：题目列表与布置动作为前端 Mock；班级列表来自 /api/classes ✅

### 7. 配置弹窗
- 提示词模板：GET/POST /api/prompt_template ✅
- NAPS2 路径、扫描输出目录、PNG 质量、扫描仪高级设置：对应 GET/POST API ✅

## 五、结论

- **核心流程完整**：扫描 → 选择文件 → OCR 识别（含译文）→ 批阅（MiniMax）→ 班级中心查看报告、学生历次记录（含作文原文与批阅内容）。
- **后端 API**：班级、文件、OCR、批阅、班级中心（含按日期报告、学生记录）、配置、历史 均已实现并对接。
- **前端**：首页、文件管理、智能阅卷、阅卷管理、班级中心、布置练习、配置 等标签页与弹窗均存在且与上述 API 或 Mock 数据对接。
- **建议**：若需“阅卷管理”“布置练习”真实数据，后续为对应接口增加后端实现并替换前端 Mock 即可。
