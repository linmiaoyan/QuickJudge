# 识别字迹并评测：模型推荐（豆包 / MiniMax 等）

## 需求

- **一步**：模型直接看图 → 识别手写 + 给出评分/评语  
- **两步**：先识别字迹（OCR/VL），再单独调用模型做作文批改  

以下推荐均可用于**英文作文**手写识别与批阅；若只做**中文**作文，可额外考虑有道智云「中文作文批改（图像识别）」API。

---

## 一、一步到位（推荐优先试）

### 1. 豆包（火山引擎）— 视觉模型

- **能力**：多模态视觉模型 **Doubao-1.5-vision-pro** 等，支持图像理解、场景分析。  
- **用法**：一次请求传「作文图片 + 提示词」，让模型同时完成：  
  - 识别图中手写英文内容（可要求输出纠错后文本 + 译文）  
  - 给出评分、评语、分项建议  
- **接入**：  
  - 火山引擎控制台创建 API Key，选择豆包视觉模型、创建推理接入点。  
  - 部分渠道提供 OpenAI 兼容接口，可直接用 `OpenAI(base_url=..., api_key=...)` 传 image url 或 base64。  
- **优点**：国产、文档全、有免费额度；一步即可完成识别+评测。  
- **文档**：火山引擎 - 视觉内容理解（Doubao-1.5-vision-pro）、豆包模型服务。

### 2. MiniMax — 多模态 Chat

- **能力**：**Chat Completions** 支持图像输入（base64），文本模型如 **MiniMax-M2.5 / M2.1** 具备图像理解能力。  
- **用法**：一条对话请求，内容里带图片 + 文字提示，例如：  
  - “请识别这张图片中的手写英文作文，先输出识别与纠错后的英文文本及中文译文，再给出评分和详细评语。”  
- **接入**：  
  - [MiniMax 开放平台](https://platform.minimaxi.com/) 创建 API Key。  
  - 消息格式示例：`"请识别并批改这篇作文: [图片base64:{img_base64}]"`。  
  - 支持 OpenAI SDK / HTTP，便于在现有 Flask 里加一个“视觉批改”接口。  
- **优点**：一步完成识别+评测；多模态文档清晰，有教育场景示例（如拍照识图）。  
- **文档**：MiniMax 开放平台 - 接口概览、拍照识图互动学习。

## 二、两步分开（识别 + 评测）

若希望「识别」和「评测」解耦（例如识别用专业 OCR，评测用大模型）：

| 步骤     | 可选方案 |
|----------|----------|
| **识别** | 讯飞 OCR（当前项目已用）、百度手写作文识别（多模态）、或豆包/MiniMax 视觉只做“从图中转写文本+纠错+译文”。 |
| **评测** | 豆包文本模型、MiniMax 文本模型、或现有 DeepSeek/SiliconFlow；输入为识别/纠错后的文本，输出评分与评语。 |

当前项目已是「讯飞 OCR → 提取学号 → 译文 → 批阅」的两步流程；若把「批阅」换成豆包或 MiniMax 的文本模型，只需改 `grade_essays()` 里调用的 `base_url` / `api_key` / `model` 即可。

## 三、豆包 vs MiniMax 简要对比

| 项目       | 豆包（火山引擎）     | MiniMax              |
|------------|----------------------|----------------------|
| 一步识+评  | 视觉模型直接看图批改 | Chat 多模态看图批改  |
| 两步       | 视觉识别 + 文本批改  | 同左                 |
| 文档/生态  | 火山引擎文档完善     | 开放平台文档清晰     |
| 免费/价格  | 有免费额度           | 按量付费，可先试用   |
| 英文作文   | 支持                 | 支持                 |

**建议**：  
- 想**最少改动、一条请求搞定**：优先试 **MiniMax 多模态 Chat**（图片+提示词）或 **豆包视觉模型**（同样图片+提示词）。  
- 想**继续用现有讯飞 OCR**，只把“批改”换成国产模型：把评测步改为豆包或 MiniMax 的**纯文本**接口即可。

## 四、与本项目的衔接

- **当前**：`/api/ocr` 讯飞识别 → 译文（translate_essay_to_chinese）→ `/api/grade` 批阅（DeepSeek/SiliconFlow）。  
- **一步方案**：新增接口（如 `/api/vision_grade`），请求体为图片 base64，调用豆包视觉或 MiniMax 多模态，返回识别结果 + 译文 + 评分 + 评语；前端“识别并批阅”可改为调该接口。  
- **两步方案**：保持 `/api/ocr`，仅将 `/api/grade` 的 `base_url` / `api_key` / `model` 改为豆包或 MiniMax 的文本端点，即可实现“识别用讯飞，评测用豆包/MiniMax”。

如需在 `app.py` 里加豆包或 MiniMax 的示例调用（一步或两步），可说明你更倾向哪种（豆包 / MiniMax、一步 / 两步），我可以按当前项目结构写出具体代码片段与环境变量示例。
